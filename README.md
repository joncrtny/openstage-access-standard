# OpenStage (working name)

**An open data standard for accessibility information in live entertainment.**

> **Status:** v0.1 draft. A proposal, not a standard. Not endorsed by any organisation. Shared to find collaborators and a steward.

## What this is

A shared way for theatres and live-entertainment venues to publish accessibility information once, in one format, that any website, app, or ticketing system can read. It covers three things: venue access, the sensory and content profile of a show, and which access services run at each performance.

## Why

Today this information is scattered across venue pages, PDF brochures, third-party listings, and box-office phone calls, in no common format and often unverified. That makes a trip hard to plan for disabled audiences, and hard for venues to keep consistent across channels. A shared standard lets one feed drive many services. Public transport did this with [GTFS](https://gtfs.org); UK sport did it with [OpenActive](https://www.openactive.io).

## What's in the repo

- [`OpenStage-Access-Standard-v0.1.md`](./OpenStage-Access-Standard-v0.1.md) — the draft scope and data model. Start here.
- *(planned)* JSON-LD examples, a data dictionary, and a validator.

## Scope (v0.1)

Theatre and live entertainment in the UK: venue access, a production's sensory and content profile, and per-performance access services. Cinema and sport may follow later as separate profiles.

## Getting involved

This needs input from people with a range of access needs, from venues, and from the services that would use the data. Useful right now:

- **Reactions to the model and the vocabularies** — open an issue.
- **Venues willing to publish a sample feed** — get in touch.
- **Anyone who can help steward it** — likewise.

To contribute, open an issue or joncrtny@gmail.com.

## Governance and licence

Developed in the open, and intended to be handed to a neutral steward rather than owned by an individual or a single company. Intended licensing: specification under Apache 2.0.

## A note on the name

"OpenStage" is a placeholder. "Open access" already refers to free scholarly publishing, so the final name will change. Suggestions welcome.
